# Інвентар Resident Evil Requiem

Вебдодаток для групової практичної роботи: каталог предметів, сітковий інвентар 8×10, сортування за площею, крафт і апгрейди зброї. Доменна модель побудована за принципами SOLID (див. `src/domain/`).

## Запуск

```bash
npm install
npm run dev
```

Відкрийте URL з термінала (зазвичай http://localhost:5173).

Збірка:

```bash
npm run build
npm run preview
```

## Механіки

- **Каталог** — усі категорії RE Requiem: зброя, споживні, боєприпаси, матеріали, ключові, апгрейди.
- **Додавання** — з урахуванням `slotSize` (зайняття клітинок); кожна одиниця — окремий слот у сітці 10×8.
- **Сортувати** — сортування за площею (width×height) і перепакування greedy first-fit.
- **Крафт** — рецепти з практичної роботи (наприклад, Трава + Хімічна рідина → Спрей, Порох + Хімічна рідина → Набої).
- **Апгрейди** — застосування `UpgradePart` до `Weapon` (ліміт слотів, лічильник апгрейдів).

## SOLID

| Принцип | Реалізація |
|---------|------------|
| **SRP** | Окремі класи `Weapon`, `Consumable`, `CraftMaterial`, `KeyItem`, `UpgradePart` |
| **OCP** | Новий тип предмета — новий клас + запис у `factories.ts` |
| **LSP** | Сервіси працюють з `Item` / `IInventoryAccess` |
| **ISP** | `IUsable`, `ICombinable`, `IUpgradeable` — не всім предметам |
| **DIP** | `CraftingService` / `UpgradeService` залежать від `IInventoryAccess` |

## Структура

```
src/
  domain/       # класи предметів
  data/         # каталог і рецепти
  inventory/    # сітка, розміщення, сортування
  services/     # інвентар, крафт, апгрейди
  ui/           # відображення
  app/state.ts  # стан додатку
```
